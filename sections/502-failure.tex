Across all 450 runs, four failure families account for the terminal blockers. \textbf{Simulation substitution} dominates at 316/450 runs (70.2\%): agents produce mock servers, C harnesses, or host-native programs that demonstrate the vulnerability pattern without running the real firmware binary. \textbf{Rehosting gap} accounts for 97/450 runs (21.6\%): agents identify the target binary but cannot configure dynamic loaders, NVRAM values, or network listeners under QEMU. \textbf{Extraction failures} (37/450, 8.2\%) and \textbf{infrastructure failures} (15/450, 3.3\%) round out the taxonomy. Terminal blockers cluster at Phase 2 (202/450) and Phase 5 (109/450).

\textbf{Extraction failures (37/450, 8.2\%).} Agents acquire firmware but misuse extraction tools or analyze binaries from PoC repositories rather than the target firmware.

\textbf{The rehosting gap (97/450, 21.6\%).} Agents identify the target binary but cannot configure dynamic loaders, NVRAM, or network listeners under QEMU, and abandon after one or two errors. Same-architecture (ARM64) devices bypass this gap via \texttt{chroot}.

\textbf{Simulation substitution (316/450, 70.2\%).} The dominant failure: agents produce mock servers, C harnesses, or host-native programs that demonstrate the vulnerability pattern but not the instance. The output may be coherent and reproducible, but wrong for the benchmark goal. Real-target gating rejects all of these.
